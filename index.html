Sistema de captura por área para el Segundo Informe de Gobierno

Este ejemplo traduce el instrumento de recolección a un flujo digital: cada dependencia captura su resumen, registra numeralia, desarrolla cada acción, adjunta evidencia, recibe comentarios de revisión y deja lista la información para su integración al informe anual.

Flujo sugerido

Captura inicial → envío a revisión → observaciones → ajustes por Secretaría → validación → integración al informe 2026.

Secretaría Carga información y evidencia por programa o acción.
Coordinación Revisa cifras, redacción y consistencia del contenido.
Integración Consolida el contenido ya validado en la versión final del informe.

1. Datos generales del reporte

Datos base para identificar a la dependencia, el responsable de la carga y el periodo del informe.

Borrador activo
Sugerencia funcional Este bloque podría vincularse al catálogo de secretarías del sistema para evitar capturas duplicadas y mantener un control por área, usuario responsable y fecha de actualización.

2. Resumen ejecutivo

Máximo 150 palabras. Debe expresar el logro principal del área de manera clara y útil para el ciudadano.

Campo estratégico
Recomendación: mantener un enfoque ciudadano, evitar tecnicismos innecesarios y resaltar el resultado alcanzado. 0 / 150 palabras

3. Datos duros y numeralia

Las acciones deben ordenarse de mayor a menor importancia. Cada fila genera su bloque de desarrollo para ampliar la explicación narrativa.

Mejora sugerida para el módulo real Puede incluir catálogos de unidades de medida, validación de montos, beneficiarios directos e incluso conexión con metas del POA para evitar inconsistencias entre áreas.

4. Desarrollo de acciones

Por cada acción capturada, se despliega un bloque para responder qué se hizo, dónde se hizo y por qué es importante.

Se actualiza automáticamente

5. Testimoniales y evidencia

Bloque para documentar caso de éxito, comentarios de beneficiarios y archivos audiovisuales que respaldan el informe.

Adjuntos de respaldo
Fotografías y comparativos

Simulación de carga de imágenes antes/después. En el sistema real, aquí se podrían almacenar carpetas o archivos vinculados a cada acción.

Videos o materiales complementarios

Espacio para clips cortos, documentos técnicos, fichas de obra, minutas o cualquier soporte adicional.

6. Comentarios y ciclo de revisión

Vista para que Coordinación o Contraloría interna deje observaciones, y el área atienda ajustes sin salir del módulo.

2 comentarios activos
Uso sugerido Cada comentario puede asociarse a un apartado, incluir responsable, fecha compromiso y estatus de atención para dejar trazabilidad de los cambios realizados.

7. Vista de integración del informe

Esta sección resume lo capturado y muestra cómo podría quedar listo para consolidarse en el Informe 2026.

Listo para consolidar

Este archivo funciona como prototipo front-end. Para operación real, convendría conectarlo a autenticación por usuario, catálogos institucionales, almacenamiento de evidencias, bitácora de cambios y un exportador que arme automáticamente los insumos del informe.
